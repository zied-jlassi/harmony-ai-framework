<good_example title="Pluralisation ICU">
**Situation**: Afficher "X points" avec grammaticalité
**Action Babel**:
1. `<thinking level="think_hard">` ICU format par langue
2. FR: `{count, plural, =0 {Aucun point} one {# point} other {# points}}`
3. EN: `{count, plural, =0 {No points} one {# point} other {# points}}`
4. AR: Format arabe avec dual + plural
5. Tester avec 0, 1, 2, 5, 100
**Résultat**: Grammaire correcte dans toutes les langues
</good_example>

### Bad Examples

<bad_example title="Texte Hardcodé">
**Situation**: Ajout rapide d'un message d'erreur
**Mauvaise Action**: `return "Erreur de connexion"`
**Pourquoi c'est mal**: Non traduisible, cassera en production multilingue
**Correction**: `return t('errors.connectionFailed')` + ajout aux 3 locales
</bad_example>

<bad_example title="RTL avec margin-left">
**Situation**: Aligner un bouton à gauche
**Mauvaise Action**: `margin-left: auto` (CSS physique)
**Pourquoi c'est mal**: Inversé en RTL, sera à droite
**Correction**: `margin-inline-start: auto` (CSS logique)
</bad_example>

<bad_example title="Ignorer Pluralisation">
**Situation**: Afficher un compteur
**Mauvaise Action**: `${count} point(s)` avec parenthèses
**Pourquoi c'est mal**: Grammaticalement incorrect, non localisable
**Correction**: ICU MessageFormat avec règles par langue
</bad_example>

<bad_example title="Namespace Plat">
**Situation**: Ajouter des traductions
**Mauvaise Action**: Tout dans `common.json` sans organisation
**Pourquoi c'est mal**: Fichier énorme, collisions de clés, maintenance difficile
**Correction**: Namespaces par module (auth.login.title, games.math.score)
</bad_example>

## Support RTL (Arabe)

### Configuration HTML

```html
<!-- Détection automatique -->
<html lang="ar" dir="rtl">
```

### CSS Logical Properties

```css
/* ❌ ÉVITER - Properties physiques */
.card {
  margin-left: 16px;
  padding-right: 8px;
  text-align: left;
}

/* ✅ UTILISER - Properties logiques */
.card {
  margin-inline-start: 16px;
  padding-inline-end: 8px;
  text-align: start;
}
```

### Mapping Logical Properties

```
┌─────────────────────────┬─────────────────────────┐
│ Property Physique       │ Property Logique        │
├─────────────────────────┼─────────────────────────┤
│ margin-left             │ margin-inline-start     │
│ margin-right            │ margin-inline-end       │
│ padding-left            │ padding-inline-start    │
│ padding-right           │ padding-inline-end      │
│ left                    │ inset-inline-start      │
│ right                   │ inset-inline-end        │
│ text-align: left        │ text-align: start       │
│ text-align: right       │ text-align: end         │
│ border-left             │ border-inline-start     │
│ border-right            │ border-inline-end       │
└─────────────────────────┴─────────────────────────┘
```

### Tailwind RTL

```html
<!-- Classes RTL avec prefix -->
<div class="ml-4 rtl:mr-4 rtl:ml-0">
  <span class="text-left rtl:text-right">Texte</span>
</div>

<!-- Ou utiliser space-* qui s'adapte automatiquement -->
<div class="flex gap-4">
  <span>Item 1</span>
  <span>Item 2</span>
</div>
```

### Hook useDirection

```typescript
import { useTranslation } from 'react-i18next';

export function useDirection() {
  const { i18n } = useTranslation();
  const isRTL = i18n.language === 'ar-SA';

  return {
    isRTL,
    dir: isRTL ? 'rtl' : 'ltr',
    startAlign: isRTL ? 'right' : 'left',
    endAlign: isRTL ? 'left' : 'right',
  };
}

// Usage
function GameCard() {
  const { dir, isRTL } = useDirection();

  return (
    <div dir={dir} className={isRTL ? 'font-arabic' : ''}>
      {/* contenu */}
    </div>
  );
}
```

```typescript
import { useTTS } from '@/hooks/useTTS';

function FeedbackMessage({ type }: { type: 'correct' | 'incorrect' }) {
  const { t, i18n } = useTranslation();
  const { speak } = useTTS();
```

## Formatage Localisé

### Dates

```typescript
import { useTranslation } from 'react-i18next';

function FormattedDate({ date }: { date: Date }) {
  const { i18n } = useTranslation();

  const formatted = new Intl.DateTimeFormat(i18n.language, {
    dateStyle: 'long',
    timeStyle: 'short',
  }).format(date);

  return <time dateTime={date.toISOString()}>{formatted}</time>;
}

// Résultats:
// fr-FR: "15 janvier 2025 à 14:30"
// en-US: "January 15, 2025 at 2:30 PM"
// ar-SA: "١٥ يناير ٢٠٢٥ في ٢:٣٠ م"
```

### Nombres

```typescript
function FormattedNumber({ value }: { value: number }) {
  const { i18n } = useTranslation();

}

// Résultats:
// fr-FR: "1 234,56"
// en-US: "1,234.56"
```
